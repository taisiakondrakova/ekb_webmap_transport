# ekb_webmap_transport
# Web-GIS: Транспортная инфраструктура Екатеринбурга

> Аналитическая веб-карта обеспеченности остановками общественного транспорта
> и велоинфраструктурой для городской администрации Екатеринбурга.
> Показывает дефициты покрытия и зоны роста — районы, где жители плохо обеспечены транспортом.

## 📋 Описание проекта

Карта создана в рамках курса «Методы пространственного анализа» (2024).
Цель — визуализировать транспортную обеспеченность семи административных районов
Екатеринбурга и выявить территории, где жители находятся вне зоны пешей доступности
от остановок общественного транспорта.

Все поля подписаны на русском языке. Пример для района:

| Поле | Описание |

## Методология

### Сбор данных
Все пространственные данные получены из OpenStreetMap через Overpass API
на сайте [overpass-turbo.eu](https://overpass-turbo.eu) с ограничением по
административной границе Екатеринбурга (`admin_level=6`). Данные экспортированы
в формате GeoJSON.

### Обработка в QGIS
- Система координат для расчётов: **EPSG:32641** (UTM зона 41N)
- Удаление дублирующихся геометрий инструментом *Delete Duplicate Geometries*
- Подсчёт остановок по районам: *Count Points in Polygon*
- Расчёт показателя обеспеченности: `stop_count / (population / 1000.0)`
- Буферные зоны пешей доступности: 400 м от остановок наземного ОТ, 800 м от станций метро
- Зоны непокрытия: разность полигона города и объединённых буферов (*Difference*)

### Картографическое оформление
- Картограмма районов: метод естественных разрывов **Jenks**, 5 классов,
  цветовая шкала YlOrRd (ColorBrewer)
- Маршруты ОТ: категоризация по полю `route` с цветовым кодированием
- Экспорт в веб: плагин **qgis2web** (Leaflet), кластеризация остановок через
  Leaflet.markercluster

## Что получилось хорошо

- Успешно выгружены все слои из OpenStreetMap через Overpass API: остановки ОТ,
  станции метро, велодорожки, маршруты, границы районов
- Реализованы все требуемые способы отображения: картограмма по районам,
  кластеризация остановок, линейный способ для маршрутов и велодорожек,
  маркеры для станций метро
- Настроены информативные попапы с русскоязычными подписями полей
  для всех ключевых слоёв
- Добавлены три дополнительных инструмента: поиск адреса, измерение расстояний,
  геолокация пользователя
- Реализована интерактивная панель управления слоями с toggle-переключателями
  и вкладками «Слои» / «О карте» — вместо стандартного Layer Control qgis2web
- Карта опубликована на GitHub Pages и доступна по открытой ссылке

## Что не получилось / что можно улучшить

- Данные OpenStreetMap по маршрутам ОТ неполны: часть маршрутов не оцифрована
  или имеет устаревшие атрибуты, что снижает точность анализа покрытия
- Население районов внесено вручную на основе переписи 2020 года — актуальные
  данные на уровне кварталов получить не удалось, что огрубляет расчёт
  обеспеченности
- Буферные зоны 400 м и 800 м не добавлены как отдельные визуальные слои —
  отображаются только итоговые зоны непокрытия
- Карта не адаптирована для мобильных устройств: панель управления перекрывает
  часть карты на узких экранах
- В перспективе стоит добавить heatmap плотности остановок и фильтрацию
  маршрутов по типу транспорта прямо из панели
